Anti-padrão

Usar duas seleções se ou não quando poderia usar seleção alternativa
if (nota < 5)
IO.println("reprovado");
if (nota >=5)
IO.println("aprovado");

Ifs aninhados são difíceis de entender. Evite-os.

Padrão escolha sequencial (um de muitos ou elif)
Quando?
– Precisa escolher uma dentre várias ações possíveis
– A escolha depende de condições separadas
– Condições podem ser verificadas sequencialmente

Padrão cálculo incremental

● Quando?
– A entrada é uma coleção de dados
– A resposta pode ser calculada incrementalmente, passo a passo, analisando um dado de cada vez

● Como? Se a entrada é formada pelos dados a1, a2, ..., an
– calcule a resposta r1 para a1
– calcule a resposta r2 para a1 e a2 usando r1
– calcule a resposta r3 para a1, a2, a3 usando r2
– No final, teremos rn, a resposta para toda a coleção
● Para r1, r2 .. rn podemos usar uma mesma variável r

Comando switch
Permite escolher uma ou mais ações dependendo da igualdade de um valor com alguma constante
O tipo do valor para a escolha pode ser: byte, short, char, and int, wrappers correspondentes, enumerados ou Strings
int quarter = ...; // any value
String quarterLabel;
quarterLabel = switch (quarter) {
case 0 -> "Q1 - Winter";
case 1 -> "Q2 - Spring";
case 2 -> "Q3 - Summer";
case 3 -> "Q4 - Autumn";
default -> "Unknown quarter";
};
